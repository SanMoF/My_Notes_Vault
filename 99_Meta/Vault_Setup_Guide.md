# Vault Setup Guide

## How to Use Your Second Brain

### 1. Daily Workflow
1. **Capture** → Drop quick notes in [[00_Inbox/Inbox|Inbox]]
2. **Process** → Move and organize inbox items
3. **Create** → Use templates for structured notes
4. **Connect** → Link related notes with `[[ ]]`
5. **Review** → Weekly review in [[60_Daily/Weekly_Reviews]]

### 2. Using Templates

To use a template:
1. Create a new note
2. Open the Command Palette (`Ctrl+P`)
3. Search for "Templates: Insert template"
4. Select the appropriate template

**Available Templates:**
- [[Course_Note]] - For class content
- [[Project]] - For hands-on projects
- [[Component]] - For cataloging parts
- [[Concept]] - For theory and explanations
- [[Formula_Sheet]] - Quick reference sheets
- [[Meeting_Note]] - Meeting notes
- [[Daily_Note]] - Daily journal

### 3. Folder Structure

| Folder | Purpose |
|--------|---------|
| **00_Inbox** | Quick capture, process daily |
| **10_Courses** | Academic content by semester |
| **20_Projects** | Active and completed builds |
| **30_Knowledge** | Evergreen technical knowledge |
| **40_Resources** | References, papers, datasheets |
| **50_Career** | Resume, skills, job search |
| **60_Daily** | Journals and daily notes |
| **99_Meta** | Templates, system files |

### 4. Recommended Plugins

Consider installing these community plugins:
- **Dataview** - For dynamic tables and queries
- **Templater** - For advanced templates
- **Calendar** - For daily notes
- **Excalidraw** - For sketches and diagrams
- **Advanced Tables** - For better table editing

### 5. Linking Strategy

- Use `[[Note Name]]` to create links
- Create MOCs (Maps of Content) for major topics
- Use tags for cross-cutting themes: `#project`, `#concept`, `#course`
- Link liberally - connections = knowledge

### 6. Naming Conventions

- **Courses:** `COURSE_CODE - Course Name` (e.g., `MECH 301 - Control Systems`)
- **Projects:** `Project - Name` (e.g., `Project - Line Follower Robot`)
- **Components:** `Component Name - Model` (e.g., `DC Motor - 12V 300RPM`)
- **Concepts:** Use descriptive names (e.g., `PID Controller Tuning`)

### 7. Weekly Review Checklist

- [ ] Process inbox to zero
- [ ] Review active projects
- [ ] Review daily notes for insights
- [ ] Update progress on courses
- [ ] Check off completed tasks
- [ ] Plan next week
